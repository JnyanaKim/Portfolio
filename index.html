Project 01

국방통합재난관리정보체계 개발사업 (2022.07 – 2024.02)

재난상황판 개발 · 이기종 솔루션 연계
Overview

국방 분야로 사업을 확장하려던 회사의 첫 국방 도메인 프로젝트였습니다. 회사의 아이덴티티인 재난예방·대응 시스템을 국방의 영역에서 구축하는 것이 목적이었습니다.
시스템의 핵심은 지휘관의 판단을 돕는 VCP와 그 근거가 되는 재난상황판이었습니다.

Mission

재난상황판 구현을 담당했습니다. 기상정보, 침수흔적도, 태풍정보, 산불정보 등 재난 발생과 관련된 데이터를 시각화하여 어떤 지역에 어떤 재난이 발생했는지 효과적으로 확인하도록 기능을 구현하는 것이 제게 주어진 과제였습니다.

Key Decisions

WebSquare 환경에서 지도 기능을 처음부터 구현하는 대신, 사업에서 활용하는 GIS 솔루션 엔진이 OpenLayers를 포함하고 있다는 점에 주목했습니다.
WebSquare 코어 개발자와 협업해 해당 엔진을 로드하는 브릿지를 확보하고, JavaScript로 OpenLayers API를 직접 호출하는 방식을 택했습니다.
GIS 경험은 없었지만 공식 문서와 사내 GIS팀의 도움으로 Point → Polygon 순으로 Source·Layer를 구성하며 4종 이상의 재난 데이터 표출을 완성했습니다.

Outcome
  • GIS 미경험 상태에서 4종 이상의 재난 데이터(기상·침수흔적도·태풍·산불 등) 시각화 기능 구현
  • 이기종 솔루션 브릿지 구현 — GIS 엔진 스크립트를 WebSquare 환경으로 이식
  • 재난작전 지휘 판단 근거 화면으로 실사용
How This Shaped My Work

새로운 기술은 혼자 해결하는 게 아니라 동료 및 기술 전문가와 협업하며 빠르게 프로젝트에 적용하는 게 중요하다는 기준을 갖게 됐습니다.
이 경험은 이후 국군의무사령부 프로젝트에 다시 함께하자는 제안으로 이어졌습니다.

Project 02 · 대표 프로젝트

국군의무사령부 통합관제시스템 구축 (2024.08 – 2026.04)

데이터 연계 모듈 개발 · 벤더 컨택포인트 · 개발 10개월 · 운영 12개월
Build
Overview

회사의 핵심 솔루션인 통합관제 솔루션을 응급의료체계에 적용하는 것을 목표로, 국방 사업을 회사의 핵심 도메인으로 확장하기 위한 신규 구축 사업이었습니다.
제 책임감과 성장성을 보고, 이전 프로젝트에서 함께했던 PM의 제안으로 참여한 프로젝트입니다.

Mission

응급의료체계 구축을 위한 병원정보·실시간 병상정보 등 요구사항에 맞춰 데이터 연계 에이전트를 개발하고, 장기 운영이 가능한 구조를 구축하는 것이 주어진 업무였습니다.

Key Decisions

연계처별로 Job을 분리해 병행 처리와 장애 격리를 동시에 확보했습니다. 하나의 연계처에서 예외가 발생해도 나머지 연계는 영향받지 않으며, 연계 로직의 예외는 에러 테이블에 적재해 사후 추적이 가능하도록 했습니다.
이중화 환경에서는 어느 서버의 어떤 연계가 문제인지 식별하는 것이 운영의 핵심이라 판단해, 연계처와 서버 IP를 키로 에이전트 상태를 주기적으로 DB에 기록하는 모니터링 로직을 별도로 구현했습니다. 운영 단계에서 실제로 이 상태 테이블을 근거로 장애 원인을 좁혀갈 수 있었습니다.
또한, 연계 에이전트 테스트 단계에서 데이터 미수신 시, 에이전트뿐만 아니라 네트워크 망 연결도 의심해야 한다는 판단으로, 체계 통합 PL이 이미 연결 완료로 보고받았던 구간까지 L4 스위치·CDS 벤더에 직접 재확인을 요청해 정책 누락을 찾아냈고 [개방망 → 폐쇄망]으로의 데이터 연계 흐름을 구축했습니다.

Outcome
  • 3개 연계처(중앙응급의료원·건강보험심사평가원 등) 통합 연계 — 일 3만 건 / 주 10만 건
  • 연계처별 Job 분리 구조로 일부 연계 실패 시에도 나머지 연계 정상 동작
  • 개방망 → 폐쇄망 데이터 흐름 구축
How This Shaped My Work

효율적으로 기능을 만드는 것과 안정적으로 운영하는 것은 함께 고려되어야 한다는 설계 기준을 갖게 됐습니다.

Operate
Overview

구축 완료 후 약 1년간 운영과 유지보수를 담당했습니다.

Mission

서비스를 안정적으로 운영하고 사용자 영향을 최소화하는 것이 가장 중요한 역할이었습니다.

Key Decisions

운영의 본질은 장애를 해결하는 것보다 사용자의 업무 지연을 막는 것이라고 판단했습니다.
그래서 장애 발생 시 가장 먼저 한 일은 원인 분석이 아니라 어떤 업무부터 살려야 하는지 정하는 것이었습니다. 서버랙 다운 장애 당시, 근무지와 현장 간 거리로 즉시 대응이 어려운 상황이었습니다. 현장 운영자와 원격으로 협업해 핵심 업무를 우선 복구하고, 이후 현장에 직접 이동해 나머지 업무 복구와 전체 운영 구조 정상화를 확인하는 순서로 대응했습니다. SSL 인증서 만료 건은 반대로 장애가 나기 전에 발견한 사례입니다. 영향받는 시스템을 먼저 추려 사용자에게 공지하고, 업무 지장이 가장 적은 일정에 재기동·갱신해 장애 자체를 예방했습니다.

Outcome
  • 12개월 운영, 장애 5건 대응 — 핵심 업무 장애 복구까지 매번 3시간 초과 X
  • SSL 인증서 만료 사전 발견 → 사용자 공지 후 업무 지장을 최소화한 일정에 재기동·갱신, 예기치 못한 장애 예방
  • 운영 절차 매뉴얼화 및 인계
How This Shaped My Work

장애 앞에서 가장 먼저 필요한 것은 기술적 원인 분석이 아니라 사용자 업무에 대한 이해라는 것을 배웠습니다. 무엇이 먼저 살아나야 하는지 알아야 제한된 시간과 인력을 올바르게 배분할 수 있고, 그 판단은 시스템을 만든 사람이 업무를 이해하고 있을 때만 가능합니다.

"좋은 시스템은 빠르게 만드는 것과 오래 운영하는 것 사이의 균형에서 완성됩니다."

Project 03

경기소방 청사관리시스템 구축 (2025.10 – 2026.06)

권한 관리 · 게시판 · 청사 에너지 관리 · E-호조 연계
Overview

소방 본부 및 서·센터 이전 사업의 발생에 따라 수기로 관리되던 청사 부지나 건물 정보, 건물 능력에 대한 기초 데이터들을 전산화하는 것을 목표로 한 프로젝트였습니다.
이전과는 달리 짧은 기간의 프로젝트였기에 일정이 타이트했지만, 소방 도메인을 오래해 온 회사의 자산과 동료를 믿고 프로젝트에 참여했습니다.

Mission

메뉴·화면 CRUD, 사용자 권한, 청사 조직 관리 등 시스템 관리 기능 전반과, 청사 에너지(수도·전기·가스) 데이터를 입력하고 연간 차트·그리드로 보여주는 기능, E-호조 시스템 연계까지 담당했습니다. 청사 관리를 위해 흩어진 업무를 하나로 모아 사용자 업무를 단순화하는 것이 목표였습니다.

Key Decisions

구조적으로 큰 판단이 필요한 이슈는 없었지만, 개발 방식에서는 실질적인 판단이 필요했습니다. 쿼리 생성이나 JsRender 템플릿 작성처럼 반복적인 단순 작업은 AI를 적극 활용해 속도를 높였고, 핵심 로직은 사내 보안 정책과 프로젝트 영향도를 고려해 AI에 맡기지 않고 직접 작성했습니다. 어디까지 AI와 협업하고 어디서부터는 직접 판단할지를 스스로 정하는 것이 중요하다는 것을 업무방식의 변화와 함께 배우게 되었습니다.

Outcome
  • E-호조 연계로 예산·사업 담당자 업무 프로세스 3단계 → 2단계 단축
  • 직무별 화면 접근 권한 관리 기능 구현
  • 반복 작업(쿼리·JsRender 템플릿)은 AI를 활용하고, 보안 영향 로직은 직접 작성 — AI 활용 범위를 스스로 판단
How This Shaped My Work

AI와 협업하는 것은 프레임워크 활용만큼 개발 생산성을 바꾸는 변화라고 생각하게 됐습니다. 다만 보안이 걸린 로직과 데이터는 여전히 사람이 판단해 지켜야 하고, AI가 대신할 수 없는 영역도 분명합니다. 결국 AI는 묻는 만큼·학습한 만큼만 답하기 때문에, 무엇을 물어야 하는지 아는 것 — 사용자와 고객을 이해하는 일이 개발자에게 더욱 중요해지고 있다는 걸 체감하게 됐습니다.

So, How I Work
01

새로운 기술과 도메인을 빠르게 이해하고 프로젝트에 적용합니다.

02

기존 자산과 공통 구조를 적극 활용해 개발 생산성을 높입니다.

03

구현뿐 아니라 운영과 유지보수까지 고려해 설계합니다.

04

문제를 혼자 해결하려 하기보다 동료와, 그 필드의 전문가와 협업해 더 좋은 결과를 만듭니다.

05

사용자와 업무를 이해하고 더 나은 서비스를 고민합니다.
