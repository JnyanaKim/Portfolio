Project 01

국방통합재난관리정보체계 개발사업 (2022.07 – 2024.02)

재난상황판 개발 · 이기종 솔루션 연계
Overview

국방 분야로 사업을 확장하려던 회사의 첫 국방 도메인 프로젝트였습니다. 회사의 아이덴티티인 재난예방·대응을 시스템을 국방의 영역에서 구축할 목적이었으며,
핵심은 지휘관의 판단을 돕는 VCP와 그 근거가 되는 재난상황판이었습니다.

Mission

재난상황판 구현을 담당했습니다. 기상정보, 침수흔적도, 태풍정보, 산불정보 등 재난 발생과 관련된 데이터를 시각화하여 어떤 지역에 어떤 재난이 발생했는지 효과적으로 확인하도록 기능을 구현하는 것이 제게 주어진 과제였습니다.

Key Decisions

GIS 경험이 없었지만 OpenLayers 공식 문서와 사내 GIS팀, WebSquare 개발자와 적극 협업하며 필요한 기술을 빠르게 습득했습니다. Point 구현에서 시작해 Polygon 등 Source와 Layer를 구현하며 맡은 기능을 완성했습니다.

Outcome
  • 재난상황판 구축 완료
  • 실제 업무 적용
  • PM에게 신뢰를 얻음
How This Shaped My Work

새로운 기술은 혼자 해결하는 게 아니라 동료 및 기술 전문가와 협업하며 빠르게 프로젝트에 적용하는 게 중요하다는 기준을 갖게 됐습니다.
이 경험은 이후 국군의무사령부 프로젝트에 다시 함께하자는 제안으로 이어졌습니다.

Project 02 · 대표 프로젝트

국군의무사령부 통합관제시스템 구축 (2024.08 – 2026.04)

데이터 연계 모듈 개발 · 벤더 컨택포인트 · 개발 10개월 · 운영 12개월
Build
Overview

이전 프로젝트에서 함께했던 PM의 제안으로 참여한 프로젝트입니다.
회사의 핵심 솔루션인 통합관제 솔루션을 응급의료체계에 적용하는 것을 목표로, 국방 사업을 회사의 핵심 도메인으로 확장하기 위한 신규 구축 사업이었습니다.

Mission

프로토타입 기반 데이터 연계 모듈을 요구사항에 맞게 완성하고, 장기 운영이 가능한 구조를 구축하는 것이 목표였습니다.

Key Decisions

프로토타입을 적극 활용해 개발 생산성을 확보했습니다. 동시에 쿼츠 스케줄러의 Job 분리, DB 기반 상태 관리, 서버 이중화를 고려해 운영 단계까지 염두에 둔 구조를 설계로 연계 서비스 모듈을 구현했습니다. 기존 자산을 활용하면서 요구 기능에 대한 커스텀으로 연계 모듈의 유지보수성과 확장성을 확보하는 데 집중했습니다.
서비스 모듈 테스트 단계에서 발생한 L4 정책 누락은 인프라 담당자와 벤더를 직접 협업해 원인을 규명했습니다.

Outcome
  • 프로토타입 재사용을 통한 개발 생산성 확보
  • 운영 가능한 연계 구조 구축
  • 확장성과 유지보수성 확보
How This Shaped My Work

효율적으로 기능을 만드는 것과 안정적으로 운영하는 것은 함께 고려되어야 한다는 설계 기준을 갖게 됐습니다.

Operate
Overview

구축 완료 후 약 1년간 운영과 유지보수를 담당했습니다.

Mission

서비스를 안정적으로 운영하고 사용자 영향을 최소화하는 것이 가장 중요한 역할이었습니다.

Key Decisions

SSL 인증서 만료를 사전에 발견해 사용자 공지와 운영부서 협업으로 장애를 예방했습니다. 운영은 장애를 해결하는 게 아니라 서비스 품질을 유지하는 과정이라는 관점으로 접근했습니다.

Outcome
  • 서비스 연속성 확보
  • 사용자 영향 최소화
  • 운영 프로세스 개선
  • PM 및 고객 신뢰 확보
How This Shaped My Work

좋은 시스템은 기능보다 운영까지 고려한 구조에서 완성된다는 업무 기준을 갖게 됐습니다.

"좋은 시스템은 빠르게 만드는 것과 오래 운영하는 것 사이의 균형에서 완성됩니다."

Project 03

경기소방 청사관리시스템 구축 (2025.10 – 2026.06)

권한 관리 · 게시판 · 청사 에너지 관리 · E-호조 연계
Overview

국군의무사령부 프로젝트에서 쌓은 운영 경험을 살려 참여한 프로젝트로, 기존 공통 프레임워크를 활용해 경기소방 청사관리 업무를 전산화하는 것을 주 목적으로 하는 프로젝트였습니다.

Mission

공통 구조를 적극 활용해 개발 생산성을 확보하고, 기관 연계를 통해 사용자 업무를 단순화하는 것이 저에게 주어진 역할이었습니다.

Key Decisions

기존 프레임워크를 적극 활용해 개발 효율을 높였습니다. E-호조 연계 구조 변경 등 요구사항 변화에도 기존 자산을 최대한 활용하는 방향으로 구현했습니다.

Outcome
  • 개발 생산성 확보
  • 사용자 업무 동선 단축
  • 유지보수 효율 향상
How This Shaped My Work

기술보다 문제를 정의하고, 기존 자산을 활용하며, 사용자에게 더 좋은 서비스를 설계하는 것이 앞으로 개발자의 중요한 역할이라고 생각하게 됐습니다. 이 프로젝트에서 AI를 적극 활용하는 동료와 대화하며, 개발자의 역할과 협업 방식이 변화하고 있다는 고민을 자연스럽게 하게 됐습니다.
